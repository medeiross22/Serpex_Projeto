Criamos um middleware chamado autenticarToken responsável por proteger as rotas da aplicação. Antes que o usuário acesse qualquer funcionalidade protegida, o middleware verifica se existe um token JWT no cabeçalho Authorization. Caso o token exista, ele é validado utilizando a chave secreta definida no arquivo .env. Se o token for válido, os dados do usuário são armazenados em req.usuario e a execução continua normalmente com next(). Caso contrário, a API retorna um erro 401 (token não informado) ou 403 (token inválido ou expirado), impedindo o acesso às rotas protegidas.
